Developer Guide

 

Pre-requisites

1.       Access to Jupyter Lab and some knowledge of how to use it

2.       GitHub account

A guide to using GitHub is outside the scope of this guide. For a beginners guide to git and GitHub, look here: https://guides.github.com/activities/hello-world/

Setting up GitHub in your Jupyter Lab space

Before you can start using GitHub in your space you need to configure it to use your GitHub account. Click the Git tab at the top of your workspace and from there open terminal.

Once the terminal is open, enter these commands:

 

jupyter-girwin3@dmjuphub01:~$ git config --global user.name "<your GitHub username>"

jupyter-girwin3@dmjuphub01:~$ git config --global user.email "<email address associated with your GitHub account>"

 

New Project

1.       Create a folder for your project

2.       Navigate into this folder

 

3.       In the top row of tabs, click on git and then Init. This will set up this folder for use with git.

 

4.       Start creating your files etc as usual with the launcher

5.       Once you are ready to save your work to GitHub, go to the side set of tabs and click on the git symbol

6.       You will see a list of untracked files. Click on the files you want to stage for adding to GitHub.

7.       You can now commit. Note at this point you are only committing locally.

8.       To commit to GitHub, create a target repo in GitHub for your project

9.       In Jupyter Lab open the git terminal in your project and add the remote repo

10.   You can now push and pull to this repo from the git pane in the side set of tabs.

Clone existing project from GitHub

1.       Open a terminal from the Git tab at top of screen

2.       Clone your repo

3.       Refresh file list and your repo will appear in its own folder

Add existing to GitHub

1.       Go into project folder in Jupyter lab

2.       In the top row of tabs, click on git and then Init. This will set up this folder for use with git.

3.       Refresh file list

4.       The Git menu in the side navigation bar will now be active you can now follow the steps from step 5 in new project

Branching

When to Branch

A branch in source control is a copy of the existing code that you can make changes to without making changes to the main code. To be accurate what happens in git is no duplication occurs but a pointer is used to track changes in that branch. You can consider this to be an under the hood type difference. The thing to understand is that git (and GitHub) is set up in a way to encourage you to use this option regularly and often .

In the context of Jupyter labs, the kind of situations where you might want to consider creating and working on a branch rather than directly in the master branch could include:

         Multiple people working on the same script

         Want to make experimental changes that if they dont work etc can be easily thrown away and you can revert back to a good working code base

         There needs to be always available a tested working script available even when you want to work on it

 

How to Branch

1.       Create a new branch using the github menu in the side nav bar

2.       Click in the + symbol to create the new branch

3.       Enter a name for your branch and click blue confirmation button

4.       Ensure that you have the correct branch selected that you want to work in.

5.       You need to either commit or discard changes before you can change branches. Changes = saved work in your notebook.

6.       If you want to push this branch to your remote server(github), open a git terminal from the top nav bar

7.       git push --set-upstream origin <name of branch>

8.       You should now see this branch with your work in your remote repo

Merge Branches

1.       Open git terminal from top nav bar

2.       Check which branch you are in: git branch

3.       We will use the git merge <branch name> command to merge that branch into your current branch

4.       If you need to change the branch you are in, use git checkout <branch name>

5.       Example sequence of commands. User starts in development branch. Changes to Master branch and then merges code in development branch into master. Commits this change and then finally pushes to remote master.

Administrator Guide

 

GitHub extension installation

install node js first(needed for any extension):

sudo -E conda install -c conda-forge nodejs

 

use sudo E for installing below:

 

jupyter labextension install @jupyterlab/git

pip install --upgrade jupyterlab-git

jupyter serverextension enable --py jupyterlab_git --sys-prefix
